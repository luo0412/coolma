### 核心设计

**数据模型**
- notes 表包含 sync_status 字段，值为 local_only | synced | pending_upload | pending_download | conflict
- 通过 local_modified 和 server_modified 时间戳比较实现冲突检测

**同步策略**
- 读取：优先本地 SQLite，本地无则拉取云端
- 创建：保存本地 → 标记 pending_upload → 异步同步到云端
- 更新：更新本地 → 记录 sync_log → 异步同步
- 冲突：标记 conflict → 弹窗让用户选择

## 批量同步修复（2026-04-11 下午）

### 问题
右上角同步按钮被禁用，SQLite 中标记为 pending_upload 的笔记无法同步到为知笔记云端。

### 根因
1. `offline/sync` action 被临时禁用（直接返回 disabled）
2. SyncService.pushToCloud 上传时传的是 `{content, title}` 纯 markdown，但为知笔记 `updateNote` API 需要 `html` 字段（用 `embedMDNote` 包装后的格式）
3. SyncService 的 `api` 适配层从 `localStorage.getItem('kbGuid')` 取值，但没有校验空值
4. NoteItem.saveToSQLite 新建 SQLite 记录时默认 sync_status 为 `local_only`（虽然 getPendingSyncNotes 也查 local_only，但语义上应该是 pending_upload）
5. NoteItem.asyncSyncToCloud 被禁用

### 修复内容
1. **`src/store/offline/index.js`** - 重新启用 `sync` action，调用 SyncService.sync() 并刷新统计数据
2. **`src/services/SyncService.js`** - 核心修复：
   - api.updateDoc: 用 `helper.embedMDNote()` 将 markdown 包装成 html，传给 WizNoteApi.updateNote 时带上 `html, title, kbGuid, docGuid, category, resources, type` 完整字段
   - api.createDoc: 同理，先创建笔记壳再上传包装后的 html
   - 添加 `getKbGuid()` 统一获取 kbGuid，所有 API 调用前校验非空
   - 移除 `electron-log` 依赖，改用 `console.xxx`（渲染进程兼容性）
   - pushToCloud/pullFromCloud 添加详细日志
3. **`src/components/ui/NoteItem.vue`** - 
   - saveToSQLite: 新建记录时也设置 `sync_status: 'pending_upload'`
   - asyncSyncToCloud: 重新启用
4. **`src/store/server/mutations.js`** - LOGIN 时已有 `localStorage.setItem('kbGuid', kbGuid)`

### 第二轮修复：2000 错误"文件夹下存在笔记，删除失败"

**根因**：`api.updateDoc` 向 `/ks/note/save` 传了 `category` 字段，值为笔记当前文件夹路径，但为知笔记误将其理解为移动操作并尝试删除旧文件夹→触发 2000 错误

**修复**：
1. `api.updateDoc` 默认不传 `category`，避免触发移动/删除文件夹逻辑
2. `pushToCloud` 只传 `title` 和 `content`，不传 `category`
3. `pullFromCloud` 重写：只拉取本地没有的笔记，已有笔记不覆盖（本地优先）
4. 修复 `api.getDocs` 中 `categoryGuid` → `category` 参数名错误
5. 修复 `execRequest` 返回值解包——API 返回已被 execRequest 解包，不需要再 `.result`
6. `pushToCloud` 跳过没有 `doc_guid` 的本地新建笔记
